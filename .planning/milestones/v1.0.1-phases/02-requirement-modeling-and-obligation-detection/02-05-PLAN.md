# Plan 02-05: Gap Closure - VCRM Attribute Parsing

**Goal:** Implement logic to parse merged VCRM table rows and populate the `VCRMAttributes` fields in the `Requirement` model, fixing the integration gap identified in the v1.0.0 milestone audit.

## Task 1: Update Requirement Initialization
- Modify `normalize_requirements()` in `src/specforge_distill/normalize.py` to inspect candidates for the `vcrm_context` flag.
- When `vcrm_context` is present, split the pipe-separated `Candidate.text` string (e.g., `REQ-001 | Test | Core safety`) and map the segments to the `method` and `rationale` fields of the `VCRMAttributes` model. Ensure the `Requirement.text` remains clean (e.g., just the ID or primary requirement text).

## Task 2: Refine Regression Test Expectations
- Update `tests/regression/fixtures/varied_docs.yaml` (specifically the "VCRM Matrix Simulation" case).
- Ensure the expected output now asserts the presence of `vcrm` attributes (e.g., `method: Test` and `rationale: Core safety`) instead of just matching the raw merged text string.

## Task 3: Verify and Fix
- Run the regression test suite (`pytest tests/regression/test_regression.py`).
- Fix any assertion errors until the newly parsed VCRM attributes correctly pass verification.
