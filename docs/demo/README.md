# SpecForge Distill Showcase

This directory contains a sample of the outputs generated by SpecForge Distill v1.0.0.

## Scenario
Imagine you have a legacy engineering specification, `system_spec.pdf`, which contains dense narrative text, architecture diagrams, and scattered requirements.

You run the following command:

```bash
distill system_spec.pdf --report
```

## The Output
The tool automatically creates the `system_spec_distilled/` directory (represented in this demo) containing four crucial files:

1.  **`manifest.json`**: The machine-readable "Digital Thread". It indexes every requirement and architecture block found, maintaining a deterministic ID, page provenance, and `interop` hooks ready for ingestion into a SysML v2 modeling tool.
2.  **`full.md`**: A consolidated, human-readable view of the entire distillation run, including any extraction quality warnings.
3.  **`requirements.md`**: A specialized view containing *only* the extracted requirements, complete with their detected Obligation levels (e.g., `SHALL`, `MUST`) and source page citations.
4.  **`architecture.md`**: A specialized view containing the non-requirement narrative blocks, which provide vital context to the systems engineers reviewing the extracted data.

### Why this matters for MBSE
By transforming a "dead" PDF into these deterministic, linked files, SpecForge Distill enables engineers to move legacy requirements directly into modern Model-Based Systems Engineering (MBSE) workflows without manually re-typing or losing traceability to the source document.
