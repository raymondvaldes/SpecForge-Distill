# Milestone v1.0.1: Stable Binary Release

**Status:** SHIPPED 2026-02-28
**Phases:** 1-4
**Total Plans:** 17

## Overview

This milestone established SpecForge Distill as a deterministic, local-first CLI for digital-text PDF distillation and then hardened the release for real end-user adoption. The original four implementation phases delivered ingestion, requirement modeling, packaging, and determinism. The shipped `v1.0.1` state also added a binary-first distribution path, stronger README/build guidance, and multi-platform release validation across Linux, macOS, and Windows.

## Milestone Summary

**Key Decisions:**

- Preserve page-level provenance as a first-class output property instead of post-processing it later.
- Keep processing local-first and deterministic as the baseline runtime model.
- Publish single-file binaries as the primary install path to remove Python friction for end users.
- Keep manifest paths relative and the manifest schema stable while packaging and docs evolve.

**Issues Resolved:**

- Added GitHub Actions packaging for Linux, macOS Intel, macOS Apple Silicon, and Windows release assets.
- Replaced an invalid release smoke-test fixture with a parseable digital-text PDF and validated real binary execution against it.
- Removed a Windows console Unicode encoding failure from CLI success output.
- Replaced an unsupported Intel macOS Actions runner with a supported one so all release targets build cleanly.

**Issues Deferred:**

- Scanned/OCR-heavy PDF support remains out of scope for the shipped release line.
- Folder and batch processing remain follow-on work after the single-document baseline.
- Richer validation/export workflows for downstream MBSE tooling remain future work.

**Technical Debt Incurred:**

- Production signing/notarization secrets still need to be configured to eliminate Gatekeeper and SmartScreen friction on official releases.
- The root `distill` wrapper remains a development convenience runner, not a universal portable interface.
