<objective>
Add the aggregate reporting layer for batch mode so users and automation clients can see which PDFs succeeded, which failed, where outputs were written, and whether the overall batch completed cleanly.

Purpose: Turn the batch execution path into a machine-readable workflow instead of a directory of loosely related per-file outputs.
Output: One documented batch summary contract, a written `batch-summary.json` artifact for normal batch runs, aligned dry-run JSON, and explicit non-zero exit behavior when any input fails.
</objective>
